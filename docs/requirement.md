## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | Amazon アフィリエイトブログ |
| サイト種別 | ブログ形式のアフィリエイトサイト |
| ジャンル | 未定（複数ジャンル対応可能な設計） |
| 収益モデル | Amazonアソシエイト（アフィリエイトリンク経由の商品購入による報酬） |
| API連携 | Amazon Product Advertising API (PA-API 5.0) |

## 2. 機能要件

### 2.1 記事管理

- Markdown/MDX で記事を作成・管理
- フロントマター（YAML）でタイトル、カテゴリ、タグ、日付、説明文を定義
- 記事内にカスタムコンポーネント（商品カード等）を埋め込み可能
- 下書き（draft）機能で非公開記事を管理

### 2.2 Amazon 商品連携

- PA-API 5.0 で商品情報を自動取得（商品名、価格、画像、評価、URL）
- ASIN を指定して記事内に商品カードを表示
- 商品情報を JSON ファイルにキャッシュ（API レート制限対策 + ビルド高速化）
- キャッシュの有効期限管理（24時間）
- API 失敗時はキャッシュからフォールバック表示

### 2.3 商品表示コンポーネント

- **商品カード**: 画像・商品名・価格・評価・アフィリエイトリンクをカード形式で表示
- **商品比較テーブル**: 複数商品をテーブルで比較表示
- **ランキングリスト**: 順位付きで商品を一覧表示

### 2.4 SEO 対策

- 各ページのメタタグ（title, description, OGP）自動生成
- 構造化データ（JSON-LD）対応（Article, Product, BreadcrumbList）
- サイトマップ（sitemap.xml）自動生成
- RSS フィード生成
- パンくずリスト
- canonical URL の設定

### 2.5 サイト機能

- レスポンシブデザイン（モバイルファースト）
- カテゴリ別記事一覧ページ
- タグ別記事一覧ページ
- 検索機能（クライアントサイド全文検索）
- ページネーション
- ダークモード切替
- 関連記事の自動表示
- ソーシャルシェアボタン

### 2.6 セール自動検知・記事生成・X 投稿パイプライン

既存のスクレイピングプロジェクト（`C:\Users\taimai\Desktop\スクレイピング`）と連携し、Amazon のセール商品を自動で検知してブログ記事を生成・公開し、X（旧 Twitter）にも自動投稿する。

#### 2.6.1 セール検知

- 日次スクレイピングで収集した商品データ（`data/amazon/{日付}-products.json`）を利用
- **割引率基準**: 商品の `discount_percent` が閾値（デフォルト 20%）以上
- **価格下落基準**: 前日の JSON と ASIN で突合し、価格が閾値（デフォルト 5%）以上下がった商品
- 両方の基準を OR 条件で適用（どちらかを満たせばセール商品として抽出）
- 閾値は `config.yaml` の `deals` セクションで設定可能

#### 2.6.2 記事自動生成

- セール商品をカテゴリ別にグルーピングして MDX 記事を自動生成
- 出力先: `affiliate-blog/src/content/blog/deals/{日付}-amazon-deals.mdx`
- 記事テンプレート: 商品名・価格・割引率・Amazon リンク（アソシエイトタグ付き）を含む
- 同日に既存記事がある場合は上書き（1 日 1 記事）
- git commit & push で Vercel に自動デプロイ

#### 2.6.3 X 自動投稿

- X API v2 + OAuth 1.0a（`tweepy` ライブラリ）で認証
- セール商品のトップ 5 件を選択してポストを組み立て
- ポスト内容: 商品名・割引率・アフィリエイトリンク + ブログ記事 URL
- 280 文字制限を考慮した文字列切り詰め処理
- 認証情報は環境変数 / GitHub Secrets で管理

#### 2.6.4 実行方式

- GitHub Actions の日次 cron（既存の `amazon_daily.yml` を拡張）
- 実行フロー: スクレイピング → セール検知 → 記事生成 → git push → X 投稿

> **IMPORTANT:** 実装詳細・設定例・クロスリポジトリ push の注意点は [`deal-pipeline.md`](./deal-pipeline.md) を参照すること。

## 6. 制約事項・前提条件

- Amazon アソシエイトプログラムへの登録が必要
- PA-API の利用には売上実績（過去30日以内に3件以上の適格販売）が必要
- PA-API のレート制限: 1秒あたり1リクエスト（初期）
- アフィリエイトリンクであることの明示が法的に必要（景品表示法・ステマ規制対応）
- Amazon ブランドガイドラインに準拠した商品情報の表示が必要
- X API Free プラン: 月 1,500 ツイート書き込み上限
- X Developer Portal でアプリ作成・OAuth 1.0a 認証キー取得が必要
- セール自動投稿は既存スクレイピングプロジェクトの GitHub Actions 上で実行
